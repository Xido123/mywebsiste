vue计算属性

2019.11.26

1.实例1

示例中声明了一个计算属性reverseMessage，提供的函数将用于属性vm.reverseMeeage的getter
vm.reverseMessage 依赖于vm.message，在vm.message发生改变的时候，vm.reverseMessage也会更新 

<div id = "app">
<p>原始字符串: {{message}}</p>
<p>计算后反转字符串: {{ reverseMessage }}</p>
<p>{{message.split('').reverse().join('')}} </p>
        </div>
<script>
 var vm = new Vue({
     el:'#app',
     data:{
        message:'Vue'
     },
     computed:{
         //计算属性的getter
         reverseMessage:function() {
             //'this'指向 vm 实例
             return this.message.split('').reverse().join('')
         }
     }
 })
</script>
2.computed vs methods

我们可以使用method来代替computed效果上两个是一样的，但是computed是基于它的依赖缓存，只有相关依赖发生改变时次啊会重新取值， 而使用methods，在重新渲染的时候，函数总会重新调用执行。可以说computed性能更好，但是如果你不希望缓存，你可以使用methods属性。 

<div id="app2">
<p>原始字符：{{ message }} </p>
<p>计算后反转的字符串：{{ reverseMessage }} </p>
<p>使用方法后反转的字符串：{{ reverseMessage2() }} </p>
        </div>
<script>
    var vm = new Vue({
        el:'#app2',
        data:{
            message:'Vue'
        },
        computed:{
            reverseMessage:function(){
                return this.message.split('').reverse().join('')
            }
        },
        methods:{
            reverseMessage2:function(){
                return this.message.split('').reverse().join('')
            }
        }
    })        
        </script>
3.computeed setter

computed属性默认只有getter，不过在需要时也可以提供一个setter

<div id="app3">
     <p>{{ site }}</p>
 </div>
 <script>
 var vm  = new Vue({
     el:'#app3',
     data:{
         name:'google',
         url:'http://www.google.com'
     },
     computed:{
         site:{
             //getter
             get:function(){
                 return this.name + '' +this.url
             },
             //setter
             set:function(newValue){
                var names = newValue.split('')
                this.name = name[0]
                this.url = names[names.length - 1]
             }
         }
     }
 })
 vm.site = "菜鸟";
 document.write('name:'+ vm.name);
 document.write('<br>');
 document.write('url' + vm.url);
 </script>
